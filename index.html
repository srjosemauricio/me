Argo CD Integration and Automation Platform

Designed and developed an automation platform to integrate GitHub Actions workflows with Argo CD operations through a REST-based automation layer.

The platform was built using Python, FastAPI and Swagger, with authentication integrated through Keycloak. It also used RabbitMQ for asynchronous processing and PostgreSQL for persistence.

  • Create Argo CD applications.
  • Create Argo CD projects.
  • Delete applications.
  • Trigger application sync operations.
  • Change project configuration.
  • Change target clusters.
  • Return confirmed logs or streaming progress back to GitHub Actions.

In the deployment workflow, GitHub Actions calls the automation API, which starts an asynchronous process. The worker checks whether the application already exists. If it exists, the process continues to sync. If it does not exist, the automation creates the application and then continues the sync flow.

Technologies: Python, FastAPI, Swagger, Keycloak, Argo CD, GitHub Actions, RabbitMQ, PostgreSQL, REST APIs, Async Processing.

GitHub to Keycloak User Internalization

Developed a worker-based automation process to synchronize GitHub group membership with Keycloak groups.

The process is triggered by a scheduled CronJob that calls the automation API. The asynchronous worker retrieves users from GitHub groups, checks whether each user already exists in the corresponding Keycloak group and, when necessary, adds the user to the correct group.

This approach helped keep identity and access management aligned between GitHub and Keycloak, reducing manual user administration and improving operational consistency.

Technologies: GitHub API, Keycloak, Python, FastAPI, CronJob, RabbitMQ, PostgreSQL, REST APIs, IAM Automation.

Dynatrace, Cluster Inventory and Confluence Automation

Created a scheduled automation workflow to support observability governance and documentation by collecting and cross-referencing information from multiple sources of truth.

The workflow runs daily and updates Confluence documentation with a consolidated view of the platform environment, including clusters connected to Argo CD, clusters registered in the ServiceNow cluster inventory and tokens stored in Key Vault.

  • Collects all clusters connected to Argo CD.
  • Collects all clusters registered in the ServiceNow cluster inventory.
  • Collects token information from Key Vault.
  • Executes cross-reference validation across the three sources of truth.
  • Compiles the results into a structured document.
  • Publishes the updated documentation automatically in Confluence.

This automation improved visibility, reduced manual documentation effort and helped identify inconsistencies between observability, inventory and access management data.

Technologies: Dynatrace, Argo CD, ServiceNow, Key Vault, Confluence, GitHub Actions, Scheduled Workflows, Automation, Documentation as Code.

Dynatrace Deployment Automation with ApplicationSets

Created an automation process to support the observability team in deploying Dynatrace-related configurations across Kubernetes clusters.

The workflow allows the observability team to add new entries to a configuration file. At runtime, the automation checks out the target branch, reads the file, generates a new Argo CD ApplicationSet configuration and applies it to AKS.

The ApplicationSet was designed to use Auto Sync, allowing the structure to be replicated automatically across all clusters listed in the configuration file.

Technologies: Dynatrace, Kubernetes, AKS, Argo CD ApplicationSet, Auto Sync, GitOps, GitHub Actions, Automation.

AI-Powered Self-Healing for GitHub Pipelines

Designed and implemented an AI automation project to provide Level 1 support for GitHub pipeline failures using autonomous AI agent squads.

The execution is triggered whenever an error occurs in any pipeline step. The system captures the execution logs and invokes a first squad of AI agents responsible for analysis. This squad identifies the programming language, maps the required tools, understands the execution context and categorizes the error.

The output returns to the workflow, which then triggers a second squad focused on resolution. The agents attempt to fix the issue up to three times during runtime.

  • If the fix works, the AI automatically opens a Pull Request with evidence of success.
  • If the issue is not resolved after the attempts, the system creates a detailed Issue for the human team.
  • At the end of the execution, the workflow publishes a summary of the analysis and result.
  • The user who triggered the workflow is notified through Microsoft Teams.

The goal of the project was to give pipelines a self-healing capability, reducing repetitive support work and accelerating the resolution of common CI/CD failures.

Technologies: AI Agents, LLMs, AutoGen, GitHub Actions, GitHub API, Pull Requests, Issues, Microsoft Teams, Logs Analysis, Pipeline Automation, Self-Healing.
